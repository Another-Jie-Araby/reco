简单手写深拷贝

codingWithQixin 4/6/2021 js面试

简单说说什么是深拷贝和浅拷贝？

  • 浅拷贝：仅仅是复制了引用，彼此之间的操作会互相影响
  • 深拷贝：在堆中重新分配内存，不同的地址，相同的值，互不影响

  • 两者就在于，浅拷贝只是简单的复制，对对象里面的对象属性和数组属性只是复制了地址，并没有创建新的相同对象或者数组。而深拷贝是完完全全的复制一份，空间大小占用一样但是位置不同。

# 下面先简单演示一下浅拷贝

const oldguy={
            name:"qixin",
            age:"22",
            friend:{
                name:"baolu",
                age:"25"
            }
        }

const newguy=oldguy
newguy.name="zitao"
console.log("oldguy.name:"+"\'"+oldguy.name+"\'") //输出oldguy.name:'zitao'
console.log("newguy.name:"+"\'"+newguy.name+"\'") //输出newguy.name:'zitao'

可以看到在把oldguy复制给newguy后，改变newguy对象中的name属性，oldguy中的name属性也会随之更改。

这是因为JS数据分为基本数据类型和引用数据类型。基本数据类型的变量存储在栈中，引用数据类型则存储在堆中，引用数据类型的存储地址则保存在栈中。

深拷贝

那么该如何改变newguy中的属性值而不改动oldguy的属性值呢

我们可以通过一个深拷贝函数来实现

# 深拷贝演示

const oldguy={
            name:"qixin",
            age:"22",
            friend:{
                name:"baolu",
                age:"27"
            }
        }

        function deepCopy(old = {}){
            if(typeof old !== 'object' || old == null){                                                                                      
                return old;                                                               
            }

            let result = {};

            if(old instanceof Array){
                result=[];
            }   else {
                result = {}
            }

            for (let key in old){
                result[key] = old[key];
            }
            return result;
        }

        const newguy=deepCopy(oldguy)
        newguy.name="zitao"
        console.log("oldguy.name:"+"\'"+oldguy.name+"\'") //输出oldguy.name:'qixin'
        console.log("newguy.name:"+"\'"+newguy.name+"\'")//输出newguy.naem: zitao

这样的话我们就可以改变newguy中的属性值而不改动oldguy的属性值啦
